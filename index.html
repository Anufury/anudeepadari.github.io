Enterprise SOPs

Battle-tested procedures for VMware and Windows Server infrastructure

ESXi Installation

  1. Boot from ISO: Insert ESXi installer USB/ISO and boot server, select installation media
  2. Disk Selection: Choose local datastore disk (clear existing partitions if needed)
  3. Network Config: Set management IP via DCUI (F2 Configure Management Network)
  4. Root Password: Set secure password meeting complexity requirements
  5. Post-Install: Enable SSH, configure NTP servers, license host, join to vCenter

vCenter Deployment

  1. Stage 1 Setup: Mount vCSA ISO, run installer UI, select "Install" deployment type
  2. Target Selection: Connect to ESXi host, specify VM name and credentials
  3. SSO Configuration: Create new SSO domain (vsphere.local) or join existing
  4. Network Settings: Assign static IP, FQDN, DNS servers, gateway
  5. Stage 2 Complete: Configure time sync, SSH access, CEIP, and verify services

HA/DRS Cluster Setup

  1. Create Cluster: Right-click datacenter New Cluster, name and enable HA/DRS
  2. HA Settings: Set admission control policy (% reserved capacity or failover hosts)
  3. DRS Automation: Configure automation level (Manual/Partial/Fully Automated)
  4. Add Hosts: Drag ESXi hosts to cluster, verify network and shared storage
  5. VM Overrides: Set per-VM HA/DRS rules for critical workloads

ESXi Patching Procedure

  1. Pre-Patch Prep: Backup vCenter database, verify VM backups, snapshot vCenter appliance
  2. Enable Maintenance: Put host in maintenance mode, vMotion VMs to other hosts, ensure DRS is enabled
  3. Apply Patches: Use vLCM or VUM to stage patches, schedule remediation during maintenance window
  4. Verify Updates: Reboot host, check ESXi version with "vmware -vl", verify hardware compatibility
  5. Exit Maintenance: Disable maintenance mode, verify HA/DRS status, document patch levels

P2V Migration (Physical to Virtual)

  1. Assessment: Document physical server specs (CPU, RAM, disk), verify VMware HCL compatibility
  2. Pre-Migration: Uninstall physical hardware agents, defragment disks, disable antivirus temporarily
  3. vCenter Converter: Deploy VMware vCenter Converter Standalone, connect to source and destination
  4. Configure Job: Select volumes to migrate, set VM hardware specs, choose datastore and network
  5. Post-Migration: Install VMware Tools, reconfigure networking, test application functionality, decommission physical server

V2V Migration (Virtual to Virtual)

  1. Source Analysis: Identify source hypervisor (Hyper-V, KVM, etc), document VM configuration
  2. Prepare Target: Ensure sufficient vSphere resources, create port groups matching source networks
  3. Conversion Method: Use vCenter Converter for Hyper-V, OVF export/import for others, or cold clone
  4. Execute Migration: Export VM to OVF/OVA, import to vSphere, or use hot migration if supported
  5. Validation: Install/upgrade VMware Tools, verify hardware version, test applications, document changes

Distributed Switch Config

  1. Create vDS: Actions New Distributed Switch, select version and uplink count
  2. Add Hosts: Right-click vDS Add/Manage Hosts, select physical adapters
  3. Port Groups: Create distributed port groups for VLANs (Management, VM Network, vMotion)
  4. Uplink Policy: Configure load balancing (Route based on IP hash/originating port)
  5. Security: Enable promiscuous mode if needed, set MAC address changes policy

VMware NSX-T Deployment

  1. NSX Manager: Deploy NSX Manager OVA, configure cluster VIP, register with vCenter
  2. Prepare Hosts: Install NSX VIBs on ESXi hosts, configure TEP (Tunnel Endpoint) network
  3. Transport Zones: Create overlay and VLAN transport zones, define uplink profiles
  4. Logical Routing: Deploy Edge nodes, create Tier-0/Tier-1 gateways, configure BGP/OSPF
  5. Security: Configure distributed firewall, create security groups, implement micro-segmentation policies

VMware Horizon VDI Setup

  1. Connection Server: Install Horizon Connection Server, configure FQDN, install SSL certificates
  2. vCenter Integration: Add vCenter Server instance, configure AD integration, set up instant clone domains
  3. Desktop Pools: Create golden image VM, install Horizon Agent, configure instant/linked clone pools
  4. App Volumes: Deploy App Volumes Manager, create AppStacks, assign applications to users/groups
  5. UAG Gateway: Deploy Unified Access Gateway for external access, configure load balancing, enable MFA

Azure Arc Onboarding

  1. Prerequisites: Create Azure resource group, register providers (Microsoft.HybridCompute, Microsoft.GuestConfiguration)
  2. Generate Script: In Azure Portal Azure Arc Add Server, generate onboarding script with service principal
  3. Deploy Agent: Run PowerShell script on servers, verify connectivity, install Connected Machine agent
  4. Enable Features: Activate Azure Policy, configure Update Management, enable Azure Monitor insights
  5. Governance: Apply tags, assign policies for compliance, configure alerts and automated responses

Windows Server Hardening

  1. Disable Services: Turn off unnecessary services (Print Spooler, Remote Registry)
  2. GPO Policies: Apply CIS benchmarks via GPO (password policies, account lockout)
  3. Firewall Rules: Enable Windows Firewall, restrict inbound to required ports only
  4. Patch Management: Configure WSUS/SCCM, auto-install critical updates
  5. Audit Logging: Enable advanced audit policies, forward logs to SIEM

vSphere Lifecycle Manager

  1. Enable vLCM: Select cluster Updates Lifecycle Manager Enable
  2. Import Image: Upload ESXi ISO to vLCM depot or use online depot
  3. Create Baseline: Define desired image with patches, drivers, firmware
  4. Remediate: Check compliance, schedule remediation during maintenance window
  5. Validate: Verify all hosts compliant, check for hardware compatibility issues
